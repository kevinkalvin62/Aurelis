# Aurelis — User Roles & Personas

Versión: 1.0

Estado: Congelado

---

# Introducción

Aurelis está diseñado para acompañar a personas que viven la música desde contextos muy distintos.

Aunque todas utilizan la misma plataforma, no todas buscan resolver los mismos problemas.

Por ello, Aurelis no define únicamente perfiles demográficos.

Define roles dentro del ecosistema musical.

Cada decisión relacionada con UX, funcionalidades, permisos o experiencia deberá considerar las necesidades específicas de estos roles.

---

# Filosofía

No diseñamos para edades.

No diseñamos para profesiones.

Diseñamos para responsabilidades.

Dos personas con la misma edad pueden utilizar Aurelis de formas completamente distintas.

Lo que determina la experiencia no es quiénes son.

Es el papel que desempeñan dentro de la música.

---

# Rol 01

## Músico Individual

### Quién es

Una persona que utiliza Aurelis para organizar su propio material musical.

Puede practicar en casa.

Preparar conciertos.

Guardar repertorio.

Crear programas personales.

No necesita colaborar con una organización.

### Objetivos

- Tener toda su música organizada.
- Transportar canciones fácilmente.
- Consultar material rápidamente.
- Practicar sin depender de otras herramientas.

### Necesidades

- Biblioteca personal.
- Programas personales.
- Sincronización.
- Acceso rápido.

### Cómo Aurelis lo ayuda

Centralizando toda su información musical en un solo lugar.

# Principios

Cada rol merece una experiencia adaptada.

No todos necesitan las mismas funciones.

La complejidad debe aparecer únicamente cuando aporta valor.

La experiencia ideal es aquella en la que cada usuario ve únicamente lo que necesita.

---

# Declaración de Permanencia

Estos roles representan la manera en que Aurelis entiende a sus usuarios.

Podrán aparecer nuevos perfiles conforme evolucione el producto.

Pero la filosofía permanecerá.

Diseñar para responsabilidades.

No únicamente para personas.
